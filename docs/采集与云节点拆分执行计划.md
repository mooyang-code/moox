# Collector and CloudNode Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 admin 中的采集管理和云节点管理拆成独立的 `moox-collector` 与 `moox-cloudnode` 服务，并支持 dataset-driven 采集任务实例生成、异步云任务和同步云函数 fan-out 调用。

**Architecture:** `admin` 保留网关与登录态；`collector` 拥有采集规则、任务实例、planner 和交易所 adapter；`cloudnode` 拥有云节点池、代码包、部署、异步 job 和同步 invocation。SCF runtime 放在 `modules/cloudnode/scf/runtime`，业务 workload 放在 `modules/collector/scf/workloads` 或 `modules/factor/scf/workloads`。

**Tech Stack:** Go, tRPC-Go, Protocol Buffers, SQLite, Tencent SCF, Vue admin gateway, MooX storage RPC.

## 阶段 5：实现异步 job 模型

**Files:**

```text
modules/cloudnode/internal/domain/async_job.go
modules/cloudnode/internal/repository/async_job_repo.go
modules/cloudnode/internal/service/asyncjob/service.go
modules/cloudnode/scf/runtime/async_loop.go
modules/cloudnode/scf/runtime/protocol/event.go
```

- [ ] 实现 `SubmitJobs`：批量写入 `t_cloud_async_jobs`。
- [ ] 实现幂等键：`owner_service + owner_ref + workload_type + payload_hash`。
- [ ] 实现 `PollJobs`：按 node pool、deployment、workload_type 拉取可执行 job。
- [ ] 实现 lease：poll 后设置 `leased_by`、`lease_deadline`、`attempt_no`。
- [ ] 实现 `ReportJobStatus`：更新 job 状态、attempt、错误信息和耗时。
- [ ] 实现过期 lease 回收：`lease_deadline < now` 的 running job 可重新分发。
- [ ] 在 SCF runtime 中实现 async loop：拉取 job、调用 workload handler、上报状态。

状态机：

```text
pending -> leased -> running -> succeeded
pending -> leased -> running -> failed
failed  -> pending when retry remains
leased/running -> pending when lease expired
```

验收标准：

```text
重复 SubmitJobs 不产生重复 job。
PollJobs 不返回已 lease 且未过期的 job。
ReportJobStatus 能记录成功、失败、超时和错误摘要。
```
