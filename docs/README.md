# MooX 架构文档

> 一站式量化金融数据平台 —— 从数据采集、存储、管理到策略回测的完整闭环

MooX 是一个面向量化金融场景的一站式数据平台，涵盖从多源数据采集、统一存储管理、到前端可视化交互的完整技术栈。项目采用 Go Monorepo 架构，通过 `go.work` 管理 10 个独立的 Go Module，配合 Vue 3 前端，构建出可单机部署也可分布式扩展的系统。

## 核心能力

| 能力域 | 说明 |
|--------|------|
| 多源数据采集 | 支持交易所（Binance 等）K 线、标的元数据采集，可扩展至行情、舆情等数据源 |
| 统一存储引擎 | 时序数据（K 线）、记录数据（标的信息）的统一存储与查询，支持列级更新 |
| 异步派生视图 | 写入主存后异步构建 OLAP 物化视图（DuckDB）和全文索引（Bleve） |
| 管理控制台 | 用户管理、权限控制、采集规则配置、包管理、云节点管理、监控 |
| 前端工作台 | 数据浏览、采集器管理、策略配置、运维监控的可视化界面 |

## 技术栈

- **后端**：Go 1.24 + tRPC-Go + Pebble + DuckDB + Bleve + NATS + SQLite + GORM
- **前端**：Vue 3.5 + TypeScript + Vite 5 + Arco Design Vue + Pinia
- **部署**：腾讯云 SCF（采集器）+ 单机/分布式 Go 服务
