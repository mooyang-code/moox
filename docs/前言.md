# 前言

## 这本书是什么

本书是 MooX 项目的架构文档合集。MooX 是一个面向量化金融场景的一站式数据平台，涵盖从多源数据采集、统一存储管理、到前端可视化交互的完整技术栈。

这份文档不是 API 参考，也不是教程，而是**设计决策的记录**——解释每个模块为什么这样设计、有哪些取舍、边界在哪里。它面向后续开发者和接入方，帮助你快速理解系统全貌。

## MooX 是什么

MooX 解决一个核心问题：**个人量化场景下，如何用一套统一的数据底座承接多种形态的金融数据。**

- 能写入实时行情、K 线、公司信息、榜单、文档、事件和因子结果
- 能支持多个市场和来源（Binance、OKX、港股、美股、A 股、文件导入、内部计算）
- 对用户暴露稳定的写入契约和查询入口，而不是暴露底层存储实现
- 在线主存、分析查询、文本检索和冷备归档职责分离，但由同一套元数据解释

## 阅读顺序

### 初次接触项目

```
架构总览 → 大仓架构 → 量化金融数据概念 → 存储概念与设计意图
```

这条线路先建立全局视角，再逐步深入存储核心概念。

### 负责存储模块

```
存储概念与设计意图 → 存储目标架构与元数据 → 协议设计 → 存储引擎架构
```

这条线路从概念到实现，覆盖存储模块的全部设计。

### 负责管理服务

```
架构总览 → 认证鉴权 → 数据库管理 → 异步任务 → 云节点管理 → 代码包管理 → 采集任务管理
```

这条线路按依赖关系排列，从基础设施到业务模块。

### 负责采集器

```
架构总览 → 协议设计 → 采集任务管理 → 云节点管理
```

采集器部署在 SCF，需要理解与管理服务的交互协议和采集规则下发机制。

## 文档约定

- 所有"全局"均指 Space 内全局，不是全系统唯一
- 表名使用 `t_` 前缀，列名使用 `c_` 前缀
- 时间格式统一使用 RFC3339/RFC3339Nano
- 写入语义为列级 patch/upsert：未携带的旧列保留
- 临时组合查询返回 `VIEW_NOT_FOUND`，不在线动态拼装
